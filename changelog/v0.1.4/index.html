    Fission v0.1.4¶

    Fission 0.1.4 is a readability and architecture release. Compared with 0.1.3, it ships a contracted dual-layer HIR presentation surface (ADR 0011), extracts the post-lift midend into dedicated owner crates (ADR 0012 Phase D), and hardens CI Fast Gate so release assets stay reproducible under layered L0–L3 gates.

    This remains a pre-1.0 patch: the documented CLI is the primary interface. NIR remains the semantic oracle; HIR is a clone-only readability polish layer that must not retarget official rankings.

    Tagged commit: 8b5d7429
    Release: https://github.com/sjkim1127/Fission/releases/tag/v0.1.4

    Release summary¶

    Area v0.1.3 v0.1.4
    Pseudocode surfaces Dual print emerging; HIR polish ad hoc Dual-layer contract: NIR oracle + HIR presentation (ADR 0011)
    HIR control sugar Limited goto/alias polish Pure if/else → select, empty-then invert, const-left commute, !(eq/ne) peel
    HIR safety No runtime structural firewall Post-pass invariants + pre-polish restore; whole-function dead-elim
    Midend layout Mostly under fission-pcode (nir/midend) fission-midend-core / normalize / structuring own substrate + free-fns
    Structuring API PreviewBuilder-heavy methods StructuringHost + free-fn owners; pcode thin residual only
    Orchestration Re-exports through pcode midend paths orchestrate / pass/structuring call fission_midend_* free-fns directly
    Fast Gate CI Multi-OS PR cost, sccache tag issues Path-filter lanes, sccache v0.10.0 tag, analysis-db on Linux tests
    Package version 0.1.3 0.1.4 (all workspace packages + lockfile)

    HIR presentation (ADR 0011)¶

    Owner: crates/fission-pcode/src/render/presentation/
    Contract: clone-only polish, same call/load evaluation count/order, no oracle swap.

    New / extended presentation passes¶

    • Pure diamond folds
    • if (c) { x = a; } else { x = b; } → x = c ? a : b (presentation-pure only)
    • if (c) return a; else return b; → return c ? a : b
    • if (c) return a; return b; → ternary return
    • x = seed; if (c) { x = a; } → x = c ? a : seed (cond/then may substitute seed for x)
    • Empty-arm cleanup
    • Strip empty else {}
    • if (c) {} else { body } → if (!c) { body }
    • Condition canonicalize
    • Const-left comparisons → var-left with flipped op (0 < x → x > 0)
    • !(x == 0) / !(x != 0) peel; Bool-only double-not peel
    • Existing polish retained: param-home alias fold, self-update fold, goto shared return expand, if/while recovery from gotos, while (1) { if (!c) break; } → while (c), single-use pure inline, cast peel, dead pure flag/temp drop

    Structural presentation firewall¶

    After the fixed-point polish, apply_hir_presentation checks structural invariants against the pre-polish tree (presentation/invariants.rs):

    Code Meaning
    use_without_def New non-param uses without any assign (over-DCE)
    call_count_increased Real call sites increased (multi-eval risk)
    load_count_increased Load sites increased
    empty_if_shell Empty if shells increased vs pre-polish

    On violation, presentation restores the pre-polish tree so broken HIR never ships. Dead pure-assign elimination uses whole-function use counts (nested subtree-local maps incorrectly dropped branch defs still live after the if).

    Validation (HIR)¶

    • Focused nextest under presentation / layered
    • scripts/check/owner_boundaries.sh (render must not import normalize/structuring)
    • External fission-benchmark local docker loop for dual-layer readability proxies (not official Pages ranking)

    Midend architecture (ADR 0012)¶

    Owner crates¶

    Crate Role
    fission-midend-core Shared IR (Hir*), action pipeline, wave stats, labels, VSA
    fission-midend-normalize Full normalize pipeline (arith, idioms, memory, recovery, types)
    fission-midend-structuring CFG facts, SESE, collapse rules, loops/switch/conditionals free-fns, StructuringHost
    fission-pcode p-code IR/optimizer, builder/orchestrate, PreviewBuilder host residual, dual-layer render, public re-exports

    Structuring free-function extraction (Phase D)¶

    • Introduce StructuringHost; production host is PreviewBuilder (host_impl.rs).
    • Free-fn owners for conditionals, loops, switch, linear body/recovery/multiblock, guarded-tail promote/canonicalize/suffix pure, SESE discovery/driver, orphan repair, collapse-rule dispatch.
    • pass/structuring and orchestrate call midend free-fns directly (structure_cfg_via_sese, build_sese_region_body, try_repair_orphan_gotos, build_linear_multiblock_body, normalize_hir_function).
    • Remove dead PreviewBuilder inherent thin wraps (loops/conditionals/switch/orphan/ linear test path). Linear tests call free-fns.
    • Residual intentionally left in pcode: p-code opcode tables, binary/type_context inventory adapters, some guarded-tail telemetry mark hooks.

    Dual-layer print ownership¶

    • Dual-layer printer lives at crate-root render/ (not under midend normalize).
    • ADR 0013: midend diagnostic format_expr_key is not the dual-layer print surface.

    Upgrade notes¶

    • No intentional CLI contract break relative to 0.1.3.
    • HIR text may change substantially (more ternaries, fewer empty else arms, fewer param homes, fewer residual gotos). NIR remains the semantic oracle for automation/benchmark case scores.
    • Consumers of internal Rust APIs should prefer:
    • fission_pcode::midend (stable re-exports), or
    • fission_midend_core / fission_midend_normalize / fission_midend_structuring for owner-specific work.
    • Install operators should pin matching fission-utils.tar.gz from the same SemVer tag when using the standalone resource bundle.
    • fission-benchmark operators: set FISSION_VERSION=v0.1.4 (release source only for official/Pages).
