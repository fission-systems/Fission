# 문서 작성 규칙

## 마크다운 스타일

1. **제목 계층**
   - H1 (`#`): 문서 제목 (하나만)
   - H2 (`##`): 주요 섹션
   - H3 (`###`): 하위 섹션

2. **코드 블록**
   - 언어 명시: ` ```rust`, ` ```bash`
   - 인라인 코드: 백틱 사용

3. **테이블**
   - 헤더 행 필수
   - 정렬 명시 (`|:---|`, `|---:|`)

## 문서 위치

| 종류 | 위치 |
|------|------|
| 기능 문서 | `docs/FEATURES.md` |
| 변경 로그 | `CHANGELOG.md` |
| 프로젝트 개요 | `README.md` |
| 아이디어 | `docs/idea/` |

## 예시 문서 구조

```markdown

간단한 설명.

## 개요

상세 설명.

## 설치

```bash
명령어
```

## 사용법

### 기본 사용

### 고급 옵션

## 참고

```

## 문서 업데이트 시

1. 새 기능 → `docs/FEATURES.md` 업데이트
2. 변경사항 → `CHANGELOG.md` 추가
3. API 변경 → 코드 문서 주석 업데이트
