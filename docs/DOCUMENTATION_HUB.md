## Recommended Split

- Keep in repository:
  - Architecture and ownership contracts
  - CI/CD and build semantics
  - Release notes and compatibility claims
- Keep in Wiki:
  - Tutorials and walkthroughs
  - Troubleshooting and FAQ
  - Onboarding runbooks

## Editing Workflow

1. Update repository docs when behavior/contracts change.
2. Update Wiki when operational guidance changes.
3. Add cross-links between both sides to avoid drift.
