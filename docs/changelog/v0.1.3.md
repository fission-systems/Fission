# Fission v0.1.3

Fission 0.1.3 is a decompiler-quality and release-operations patch. Compared
with 0.1.2, it improves register and stack function-pointer call recovery on
x86/x64 and x86-32, tightens return and condition handling around CallInd and
BranchInd, lands the typed program-metadata substrate, and hardens the
conservative release-tag gates used to publish SemVer assets.

This remains a pre-1.0 patch: the documented CLI is the primary interface, and
pseudocode can continue to improve as recovery rules tighten.

## Release summary

| Area | v0.1.2 | v0.1.3 |
| --- | --- | --- |
| Function-pointer calls | Register CallInd often opaque / incomplete | Opaque cast-callable form for register CallInd; ABI arg recovery |
| Tail calls | BranchInd fp tails mostly ARM-oriented | x86/x64 `jmp reg` tail-call recovery with staged arg peel |
| x86-32 cdecl | Stack arg / return binding gaps | ESP-staged args, frame restage, EAX call-result binding, stack-param freeze |
| Return join | Pre-call EAX arg could win over call result | Diamond join prefers live call-result binding |
| Null-check conditions | `test reg` after redef could surface wrong param | Flag-test peel stops at tested register; freeze Copy source at def site |
| Program metadata | Loader facts only | Typed `fission-analysis-db` substrate wired into decompiler preflight |
| CI / release | Standard lanes | L0 Fast + L1 Heavy on main, L2 Release E2E before tag, L3 CD with utils bundle |
| Package version | `0.1.2` | `0.1.3` (all workspace packages + `fission_cli --version`) |

## Decompiler quality

### CallInd and BranchInd (function pointers)

- Lower register-held CallInd as cast function-pointer calls rather than bare
  symbols or undeclared targets.
- Prefer call-result bindings over pre-call argument writes to EAX/RAX at
  diamond joins (null → 0, non-null → `(*(fp))(…)`).
- Recover x86/x64 BranchInd through a register-held function pointer as a tail
  call with ABI argument recovery from the BranchInd block and a single
  predecessor when needed.
- When prefer_source recovers tail-call args, peel same-block Copy/ZExt so
  staged values (`mov ecx, edx; zext rcx`) surface as the source param, not the
  destination ABI slot.

### x86-32 cdecl CallInd / BranchInd

- Include X86_32 in call-result register recovery so CallInd materializes
  `eax = (*(fp))(…)` and returns bind that result.
- Recover cdecl arguments staged as Stores to `[esp+k]` (k ≥ 0) before CallInd,
  not only push-below-ESP forms; suppress those stores as call-arg setup
  (avoids undeclared `local_0` from staging).
- Recover frame restage Stores to `[ebp+8+4*i]` before BranchInd/CallInd for
  O2-style tail calls; suppress param reassignment materialization.
- Freeze staged-store RHS to the **incoming stack-param surface** reached by
  the Copy/Load chain at the staging site so later EAX reloads of the call
  target do not rewrite earlier args (`param_2` stays `param_2`).

### Conditions and related recovery

- For x86 branch predicates from `test reg`, peel unique temps to the tested
  register but do not continue through register-to-register Copy (avoids
  `mov rax, rcx; mov ecx, edx; test rax` becoming a null-check on `param_2`).
- Lower tested values for branch predicates with Copy sources frozen at the
  defining site.
- Additional loop-carried, cmov, setnz/neg, and flag-recovery CFG fixes landed
  between 0.1.2 and 0.1.3 and are covered by crate nextest.

### Typed program metadata

- Add and wire `fission-analysis-db` as the immutable typed program-metadata
  snapshot consumed by decompilation preflight (ADR-aligned ownership).

## CI / CD and packaging

- Conservative release gates: no auto-tag on every main push; **Release Tag
  (CI green)** requires L0 Fast (push) + L1 Heavy on the SHA, then L2 Release
  E2E, then annotated tag → L3 CD.
- Heavy main-push runs the release-critical crate set (extended suite on
  nightly/dispatch only).
- CD publishes platform CLI archives (each with verified `utils/`) and a
  standalone `fission-utils.tar.gz` on the SemVer release.

## Validation notes (local quality loop)

Local fission-benchmark docker against a local Fission build (not official
Pages / `results/latest.json`) measured semantic recovery for
`advanced_patterns` / `apply_binop`:

| Compiler variant | Status after 0.1.3 work |
| --- | --- |
| gcc -O0 | 6/6 |
| gcc -O2 | 6/6 |
| gcc-m32 -O0 | 6/6 |
| gcc-m32 -O2 | 6/6 |

Host semantic harness for m32 `uint` formals widens pointer-carrying formals
for host recompilation (arm64 cannot use `-m32`); that adaptation lives in the
fission-benchmark runner and does not change decompiler surface types.

## Upgrade notes

- No intentional CLI contract break relative to 0.1.2.
- Pseudocode for function-pointer and x86-32 stack-call patterns may change
  shape (more cast-callable calls, fewer opaque branchind/local_0 artifacts).
- Operators installing from GitHub Release should pin matching
  `fission-utils.tar.gz` from the same SemVer tag when using the standalone
  resource bundle.
