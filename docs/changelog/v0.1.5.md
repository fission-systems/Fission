# Fission v0.1.5

Fission 0.1.5 is a **determinism and performance** release. Compared with
0.1.4, it closes out a multi-pass determinism-hardening effort (fixed-seed
hashing across the normalize/structuring midend, wall-clock budgets replaced
with deterministic call/work counters) and lands a ten-round performance
sweep across the decompile pipeline ā€” SLEIGH decode, structuring, and the
`--all` batch CLI path ā€” driven by profiling rather than guesswork, with
every fix validated for byte-identical output before landing.

This remains a pre-1.0 minor release: the documented CLI is the primary
interface. **NIR remains the semantic oracle**; HIR is a clone-only
readability polish layer.

Tagged commit: `2cf042d0`
Release: https://github.com/fission-systems/Fission/releases/tag/v0.1.5

## Determinism hardening

Two independent classes of nondeterminism were root-caused and fixed:

1. **Unsorted hash-collection iteration feeding a "first match wins" pick.**
   `fission-midend-normalize`'s `variable_merge.rs::collect_direct_copies`
   returned a fully-qualified `std::collections::HashSet` (missed by an
   import-based `FxBuildHasher` sweep since it wasn't a bare `use` statement),
   feeding a union-find merge where same-tier temp-prefixed names had no
   secondary tiebreak ā€” the root cause of a long-standing ELF/aarch64
   nondeterminism repro (`control_flow_gcc-elf_x64_O0`'s `main`,
   `control_flow_gcc-aarch64_O0`'s `__dcigettext`). Fixed by converting
   fully-qualified `std::collections::HashMap`/`HashSet` references to the
   crate's `FxBuildHasher`-backed aliases throughout the crate (13 files).
2. **Wall-clock (`Instant::now()`) budgets whose completion point depended on
   machine speed, not just input.** `SESE_REGION_PROOF_BUDGET_MS` and the
   shared `IfLoweringBudget`/`loops.rs` 5000ms budget were replaced with
   deterministic call-count / total-work-unit proxies
   (`SESE_REGION_PROOF_BUDGET_CALLS`, `STRUCTURING_TOTAL_WORK_BUDGET`).

Both repros are now 30/30 uniform across repeat runs (previously ~50/50 and
3+ distinct outputs across 5 runs, respectively).

## Performance: ten-round profile-driven sweep

Each round below started from a `sample`-based profile of a real corpus
function, root-caused the dominant cost, fixed it, and verified byte-identical
output before landing ā€” full detail per round is in `PROJECT.md`.

- **Round 1ā€“2**: fast local regression tooling
  (`scripts/quality/golden_corpus_check.py`) and the `quick-release` Cargo
  profile, so future rounds don't need a full `--release` rebuild per
  iteration.
- **Round 3**: `RuntimeFrontendArtifacts`/`RuntimeSleighFrontend.compiled`
  deep-cloned the whole compiled SLEIGH grammar tree under a `Mutex` on every
  cache access ā€” `Arc`-wrapped so a hit is a refcount bump. Corpus-wide mean
  `decomp_sec` 0.60s ā†’ 0.18s. `RegisterNamer::hw_name_at`'s SLA-map fallback
  did an O(map size) linear scan per call ā€” indexed by offset.
- **Round 4**: `prove_loop_carried_register_update`'s BFS walked the whole
  containing loop body uncached on every call ā€” memoized per builder
  lifetime.
- **Round 5**: the entire `fission-pcode::midend::cspec` submodule (register
  model / cspec / pspec loaders) bypassed the crate's `FxBuildHasher` alias
  entirely; `RegisterModel::name_for`'s covering-register fallback had the
  same linear-scan shape as round 3's `RegisterNamer` fix. `accumulate_pairs`:
  6.06s ā†’ 1.05s (~5.8x).
- **Round 6**: `DefinitionDependencyMap::address_contributors` rebuilt an
  O(V+E) reverse-dependency graph from scratch on every pointer dereference
  in a function instead of once for the whole walk. `bounded_tlv_sum`: 3.13s
  ā†’ 1.15s (~2.7x).
- **Round 7**: SLEIGH's per-instruction decode hot path checked several
  diagnostic env vars via raw `std::env::var(_os)` syscalls on every
  instruction; `--all` batch rebuilt the whole-binary `FactStore` (FID
  signature matching + name facts) once *per function* instead of once per
  binary. `advanced_patterns_gcc_O2.exe` (73 functions) `--all`: user CPU
  6.4s ā†’ 3.5s (~45%).
- **Round 8**: skipped a per-task watchdog-thread spawn in `--all`'s worker
  pool when no `--timeout-ms` is set. Measured impact was inconclusive on
  this platform ā€” recorded honestly rather than claimed as a win.
- **Round 9**: systematic sweep of ~19 more uncached diagnostic env-var
  checks across `fission-pcode`/`fission-midend-normalize`/
  `fission-midend-structuring`. Caught and fixed a self-inflicted `OnceLock`
  reentrancy deadlock from an earlier mechanical refactor before it landed
  (root-caused via `lldb` thread backtraces).
- **Round 10**: memoized a predecessor-graph walk
  (`live_call_result_binding_from_predecessors_for_return_register`) that
  re-explored shared-ancestor blocks once per incoming branch instead of
  reusing results ā€” a real algorithmic soundness fix, though not the
  dominant cost for the specific function that surfaced it.

## `normalize` pass-framework migration (Track B)

Continued migrating `fission-midend-normalize`'s legacy driver chains onto
the `ActionGroup`/`Pass` framework: `deadcode_dynamic` (complete, legacy
driver deleted), `proto_recovery` (5/6 chains), `stackstall` (11/12 chains),
`block_structure_1` (complete), plus SCCP, CSE, `conditional_const`,
`constant_ptr_recovery`, and `entry_param_promotion`. Two chain-selection
pitfalls documented in `PROJECT.md` for future migration slices
(`cleanup_pass` vs `fn_pass` budget-gating; diag/perf-forwarding through
callee passes).

## Upgrade notes

- **No intentional CLI contract break** relative to 0.1.4.
- Decompiled output (NIR and HIR) is byte-identical to 0.1.4 for the entire
  golden corpus and the hand-curated 6-function regression set ā€” this release
  is a performance/determinism release, not a quality-changing one. The two
  ELF/aarch64 determinism repros fixed in this release now produce a single,
  consistent output rather than one of several possible outputs; treat that
  as the only case where observed output for those specific functions may
  differ from a pre-0.1.5 run.
- `--all` batch consumers should see lower wall-clock and CPU time per
  invocation, especially for binaries with many functions; no flag or output
  shape changes.
- New `FISSION_RUST_DECOMP_WORKERS=1` / `FISSION_RUST_DECOMP_STACK_MB` env
  vars are pre-existing tuning knobs for the `--all` worker pool, unchanged
  by this release.
- fission-benchmark operators: set `FISSION_VERSION=v0.1.5` (release source
  only for official/Pages).
