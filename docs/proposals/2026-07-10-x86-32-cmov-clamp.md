# X86-32 CMOV Clamp Recovery

## 1. Baseline Row Anchor

- Binary: `corpus/dev/binaries/control_flow_gcc-m32_O2.exe`
- Function: `clamp` @ `0x4015d0`
- Before: `edx = param_1; return param_2;` (both cmovs dropped / collapsed)
- Assembly: `cmovle` then `cmovl` after early epilogue

## 2. Owner Proof

1. **Builder / CFG**: SLEIGH encodes cmov as absolute code-space `CBranch` to the next instruction, not a relative p-code delta. Intra-block skip recovery only handled `space_id==0` constants.
2. **Builder / materialize**: CBranch condition lowering lacked `current_lowering_site`, so reused unique temps resolved to later cmov defs.
3. **Builder / return recovery**: Last def of EAX was a guarded cmov Copy; inlining its RHS always returned the last source (lo).
4. **Builder / binding**: Primary return register writes each got a fresh temp; cmov default+override could not compose.
5. **Normalize / variable_merge**: Conditional copies inside `if` were treated as unconditional merge edges, coalescing accumulator with `lo`.
6. **Normalize / ordering**: Param alias copies could end up after first use; hoist pass restores dominance.

## 3. Invariant

```text
Absolute same-block forward branches are instruction-local skips (cmov).
Guarded register redefs of the return value share one binding.
Only unconditional var copies establish variable-merge identity.
Pure `v = param_N` must dominate its first use.
```

## 4. Validation

- Unit: `same_block_forward_tests`, `normalize_preserves_cmov_clamp_*`, `variable_merge`
- Real decomp: m32 O2 clamp returns composed hi/value/lo chain; count_bits still correct
