# Welcome To Fission Discussions

Fission is a Rust-based reverse-engineering and decompilation workspace that is gradually moving from a Ghidra-backed legacy path toward a Rust-owned decompiler core.

This Discussions space is the best place for:

- architecture questions
- reverse-engineering workflow ideas
- decompiler quality examples
- benchmark and corpus suggestions
- plugin and extension proposals
- UX feedback for the desktop app

## Useful Discussion Categories

### Show and tell

Share interesting binaries, preview output, comparison cases, or decompilation quality examples.

### Design / architecture

Use this for discussion around:

- NIR / HIR design
- structuring and normalization strategy
- fallback taxonomy
- type/symbol propagation
- plugin architecture

### Q&A

Use this for setup questions, build issues, usage questions, or repository navigation.

### Ideas

Use this for product ideas, AI workflow ideas, plugin concepts, and long-term roadmap suggestions.

## Discussion Guidelines

- Include binary/function context when reporting decompilation quality issues
- Prefer concrete examples over general statements
- If possible, include expected output, current output, and why the current result is problematic
- Keep scope focused so threads are easier to turn into actionable issues

## Current Reality

Fission is still under active development. Some docs are still being translated or cleaned up for public use, and some internal workflows are still evolving.

That is fine. The point of opening the project now is to make the architecture, progress, and direction visible while the engine is actively taking shape.
